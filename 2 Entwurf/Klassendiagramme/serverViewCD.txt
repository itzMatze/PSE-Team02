
Dieses Klassendiagramm zeigt die Klassen der Schnittstelle des Servers. Diese Klassen stellen die Funktionalitäten für die Kommunikation zwischen Client und Server bereit.




Plant UML code:

@startuml

title Server Controller - Class Diagram

class UserController {
  +ResponseEntity<UserProfile> getUserById(string userId)
  +ResponseEntity<void> createUser(Credentials credentials)
  ..for admins..
  +ResponseEntity<void> deleteUser(string userId)
}

class GroupController {
  +ResponseEntity<Group> getGroupById(string groupId)
  +ResponseEntity<Group[]> getGroupsByPrefferences(Prefferecnces prefs)
  +ResponseEntity<void> createGroup(Group group)
  ..for admins..
  +ResponseEntity<void> deleteGroup(string groupId)
}

class MembershipController {
  +ResponseEntity<void> addUserToGroup(string userId, string groupId)
  +ResponseEntity<void> removeUserFromGroup(string userId, string groupId)
}

class MensalineController {
  +ResponseEntity<MensaData> getMensaData()
}

class AuthenticationControler {
  +ResponseEntity<Token> login(Credentials credentials)
}

class UserProfileController {
  +ResponseEntity<void> updateUserProfile(UserProfile user)
}

@enduml
